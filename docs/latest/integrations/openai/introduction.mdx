DokuMetry SDKs (`dokumetry`) empowers you to effortlessly track and monitor OpenAI usage data and metrics from your Python/NodeJS code. 

## Get Started 

The integration is compatible with OpenAI Python SDK version `>=1.13.0` and NodeJS SDK `>=4.28.0`

<Tabs>
  <Tab title="Python">
    ```shell
    pip install dokumetry
    ```

    ```python
    from openai import OpenAI
    import dokumetry

    client = OpenAI()

    dokumetry.init(llm = client, doku_url = "YOUR_DOKU_INGESTER_URL", api_key = "YOUR_DOKU_TOKEN")
    ```
  </Tab>
  <Tab title="Javascript">
    ```shell
    npm install dokumetry
    ```

    ```javascript
    import OpenAI from "openai";
    import DokuMetry from 'dokumetry';

    const client = new OpenAI();

    DokuMetry.init({llm: client, dokuUrl: "YOUR_DOKU_INGESTER_URL", apiKey: "YOUR_DOKU_TOKEN"})
    ```
  </Tab>
</Tabs>

## Supported Endpoints

The following OpenAI Endpoints are supported for monitoring with DokuMetry

- [Chat completions](/latest/integrations/openai/examples/chat)
- [Image Create](/latest/integrations/openai/examples/image#create-image) and [Image Variations](/latest/integrations/openai/examples/image#create-image-variation)
- [Audio Speech Create](/latest/integrations/openai/examples/audio)
- [Embeddings](/latest/integrations/openai/examples/embeddings)
- [Fine Tunings](/latest/integrations/openai/examples/fine-tuning)
- [Completions](/latest/integrations/openai/examples/completions)

## Parameters

<Tabs>
  <Tab title="Python">
  | Parameter         | Description                                               | Required      |
  |-------------------|-----------------------------------------------------------|---------------|
  | llm               | OpenAI Object to track                                    | Yes           |
  | doku_url          | URL of your Doku Ingester                                 | Yes           |
  | api_key           | Your Doku API key                                         | Yes           |
  | environment       | Custom environment tag to include in your metrics         | Optional      |
  | application_name  | Custom application name tag for your metrics              | Optional      |
  | skip_resp         | Skip response from the Doku Ingester for faster execution | Optional      |
  </Tab>
  <Tab title="NodeJS">
  | Parameter         | Description                                               | Required      |
  |-------------------|-----------------------------------------------------------|---------------|
  | llm               | OpenAI Object to track                                    | Yes           |
  | dokuUrl           | URL of your Doku Ingester                                 | Yes           |
  | apiKey            | Your Doku API key                                         | Yes           |
  | environment       | Custom environment tag to include in your metrics         | Optional      |
  | applicationName   | Custom application name tag for your metrics              | Optional      |
  | skipResp          | Skip response from the Doku Ingester for faster execution | Optional      |
  </Tab>
</Tabs>

## Advanced Usage
When using DokuMetry in Production environments, It is recommened to set Skip response parameter to `true` for faster porcessing. 

To filter your LLM Usage according application or the environment, set the Application name and Environment paramaters with the actual values.
